Location Quick Reply ¶

Location sharing in Direct Messages provides users the ability to share a location as part of a conversation, as well as enable businesses to prompt users to share a location via a new type of quick reply.

Users have the ability to select from one of three choices for sharing a location in the user experience:

  • Current location (lat/long from mobile device)
  • Pinned location (lat/long selected from dragging a pin on a map)
  • Place or POI (Twitter Place ID)

In the response, developers will receive either a coordinate or place ID, which can be hydrated from GET geo/search . Developers can also send a location card to a user as part of a DM conversation by attaching a valid Twitter place ID or coordinates to a DM POST request.

Working with Location Quick Replies:

  • Requesting location from a user with a quick reply
  • Sending a location
  • Receiving a User Response via Webhook

Requesting location from a user with a quick reply ¶

Quick Reply Object ¶

quick_reply.type (required) Must be set to location
quick_reply.location.metadata (optional) Metadata that will be provided in user response payload. String, max length of 1,000 characters including spaces.

Example Request with Location Quick Reply ¶ 

{
  "event": {
    "type": "message_create",
    "message_create": {
      "target": {
        "recipient_id": "2001",
      },
      "message_data": {
        "text": "I’m sorry your burrito is bad. Which location did you order from?",
        "quick_reply": {
          "type": "location",
          "location": {
            "metadata": "external_id"
          }
        }
      }
    }
  }
}

Sending a location ¶

Quick Reply Object ¶

type (required) String value of shared_coordinate or shared_place
shared_coordinate (optional) shared_coordinate
shared_place (optional) shared_place

Shared Coordinate Object ¶

coordinates.coordinates (required) Array representation in [longitude, latitude]
coordinates.type (required) String value that must always be “Point”

Shared Place Object ¶

place.id (required) A place ID. See places documentation for full details.

Example Request with Attached Shared Coordinate ¶ 

{
  "event": {
    "type": "message_create",
    "message_create": {
      "target": {
        "recipient_id": "844385345234"
      },
      "message_data": {
        "text": "Here's my location",
        "attachment": {
          "type": "location",
          "location": {
            "type": "shared_coordinate",
            "shared_coordinate": {
              "coordinates": {
                "type": "Point",
                "coordinates": [-122.443893, 37.771718]
              }
            }
          }
        }
      }
    }
  }
}

Example Request with Attached Shared Place ¶ 

{
  "event": {
    "type": "message_create",
      "message_create": {
        "target": {
          "recipient_id": "844385345234"
        },
        "message_data": {
          "text": "Here's my location",
          "attachment": {
            "type": "location",
            "location": {
              "type": "shared_place",
              "shared_place": {
                "place": {
                  "id": "123456"
                }
              }
            }
          }
        }
      }
    }
  }
}

  **Note**

  Not all place IDs are discoverable with `GET geo/search </rest/reference/get/geo/search>`_ .

  To retrieve all available details for a place ID you may also use `GET geo/id/:place_id </rest/reference/get/geo/id/place_id>`_.

Receiving a User Response via Webhook ¶

Similar to other quick replies requests, user responses to location requests are consumed via webhook. When a user sends a location, the quick_reply_response.type property will be location. The location data will be in the attachment payload as a shared_coordinate or shared_place .

Message Create Event with Shared Coordinate ¶ 

{
  "direct_message_events": [
    {
      "type": "message_create",
      "id": "102",
      "created_timestamp": "4893199340",
      "message_create": {
        "target": {
          "recipient_id": "2000",
        },
        "sender_id": "2001",
        "message_data": {
          "text": "Here's my location https://twitter.com/i/location?lat=37.771718&long=-122.443893",
          "quick_reply_response": {
            "type": "location",
            "metadata": "external_id"
          },
          "attachment": {
            "type": "location",
            "location": {
              "type": "shared_coordinate",
              "shared_coordinate": {
                "coordinates": {
                  "coordinates": [-122.443893, 37.771718],
                  "type": "Point"
                }
              }
            }
          }
        }
      }
    }
  ]
}

Message Create Event with Shared Place ¶ 

{
  "direct_message_events": [
    {
      "type": "message_create",
      "id": "102",
      "created_timestamp": "4893199340",
      "message_create": {
        "target": {
          "recipient_id": "2000",
        },
        "sender_id": "2001",
        "message_data": {
          "text": "Here's my location https://twitter.com/i/location?place_id=123456",
          "quick_reply_response": {
            "type": "location",
            "metadata": "external_id"
          },
          "attachment": {
            "type": "location",
            "location": {
              "type": "shared_place",
              "shared_place": {
                "place": {
                  "id": "123456"
                  // ... hydrated place fields
                }
              }
            }
          }
        }
      }
    }
  ]
}
