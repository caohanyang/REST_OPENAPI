Buy APIs Overview

Topics

  • Post order tasks

Buy APIs

The following lists the Buy APIs and their capabilities:

  • Marketing API - Returns products, which lets shoppers discover new products and assists shoppers with deciding which product to purchase
  • Browse API – Lets shoppers search for specific items, retrieve the details of an item, and use eBay legacy API, such as Finding and Trading, item Ids to retrieve item details
  • Feed API – Returns the feed file of eBay items for a specific category and marketplace
  • Order API – Lets shoppers pay for items (for both guest and eBay member buyers) and lets sellers retrieves purchase order details (only for eBay member purchase orders)

The following section contains an overview of each Buy API.

Marketing API

The Marketing API provides the ability to surface eBay products based on a metric, such as Best Selling. It provides information such as, the eBay product Id (EPID), review ratings, market price, ect. This API helps shoppers discover products and motivate them to purchase items.

Browse API

The Browse API has two resources; item and item_summary

You use the item resource to:

  • Retrieve a group of items (such as a shirt that comes in various colors and sizes).
  • Retrieve the details of a specific item (such as a red shirt size L).
  • Retrieve a group of item or the details of a specific item using the legacy item Id. Legacy Ids are returned by eBay traditional APIs, such as the Trading API or Finding API . For details, see Legacy API compatibility .

You use the item_summary resource to:

  • Find items on the eBay site by keywords and or category Id. You can use the Commerce Taxonomy API to discover categories that will focus your search more effectively.
  • Filter the result set by item aspects, such as Brand
  • Apply filters to control the items returned
  • Retrieve aspect, buying option, category, and condition refinements, which you can use to create histograms so your buyers drill down on each refinement narrowing down the search domain and the results.

The Search capability has a rich set of filters. There are several URI parameters that filter the results and a large set of field filters. See the Search for Items call for information about the URI filters and see Search for Items Call Field Filters Usage Details for the details about using the field filters.

Feed API

To use this API, you must join the eBay Partner Network.

You use the Feed API to retrieve the contents of or download a feed file containing eBay items. This lets you quickly populate your site with eBay’s inventory and allows you to curate the inventory offline.

The feed files are generated daily by eBay with items that were added by top rated sellers to the eBay site in the last 24 hours. You can retrieve files up to 6 days in the past. To download all active listings, you need to download the feed files for 7 days. The items returned are based on the eBay marketplace, category, and file date specified in the call.

The first line of the feed file is the header, which indicates the order of the values on each line. The header is shown below:

ItemID, Title, Description, ImageUrl, CategoryId, Category, Color, SizeType, Size, Material, AgeGroup, Gender, SellerUsername, SellerFeedbackScore, SellerFeedbackPercentage, GTIN, Brand, MPN, ConditionId, Condition, PriceValue, PriceCurrency, ItemEndDate

For a description of each header see Item Feed file response . For an example of this file, see item feed file example .

Order API

The Order API has two capabilities; Checkout and Order.

You use the guest_checkout_session and checkout_session resources to:

  • Change the shipping address and the payment method for a purchase order.
  • Change the quantity and shipping method for specific line items.
  • Create a purchase order and submit it for payment.

Important! For guest checkout, your application must comply with PCI security standards to ensure proper handling of sensitive credit card data.

You use the guest_purchase_order and purchase_order resources to:

  • Retrieve the details of a purchase order
  • Check the payment status of an order

Post order tasks

All post order tasks, such contacting the seller, returning the item, or filing a dispute, are performed on the eBay site and handled by the seller. When a buyer purchases item(s) using the Buy APIs, eBay sends the buyer an order confirmation email for each seller. This email contains a View order details link and an access code. The buyer clicks on the link and then signs into eBay as a guest using their email address and access code. From there they can contact the seller, report that they didn't receive the item, or request to return the item.

Legacy API compatibility

There are differences between how legacy APIs, such as Finding , Trading , and Shopping , and the Buy RESTful APIs, such as Browse, return the identifier of an "item". There is also a difference in what the item Id represents and in the format of the item Id value returned.

What an item Id represents

The legacy APIs return listing Ids. A listing can be for one specific item, which can have many in stock. Or the listing can be a group of the same item that has variations, such as a shirt that comes in several sizes and colors. The legacy APIs do not return the Ids of the individual items within a group, such as the red shirt size M and the red shirt size L. The legacy APIs return this value in the ItemID field but this Id represent the Id of a listing.

The RESTful APIs return either the Id of a group or the Id of an item, which can be a non-variation item or a specific item in an item group. These values are returned in the itemGroupId and itemId fields respectively.

Item Id formats

The format of the item Id from the legacy APIs is different than the format from the RESTful APIs. The differences are shown in the following table.

Legacy API Item Id Format

RESTful API Item Id Format

Listing Id: 262446562346

Specific item: v1|112330214619|0

Specific item in a group: v1|1234567890|3344112255

Item Group Id: 151915076499

Important! The RESTful format should be treated as an opaque string, which should not be decomposed or constructed, since it is subject to change without notice.

Although the Ids returned by the legacy APIs cannot be used directly with the RESTful APIs, the following details how you can use a legacy item Id to obtain the RESTful item Id.

Trading and Shopping API item Ids

The Trading and Shopping API responses include the following fields. You use these fields to determine which Browse API call to use to get the RESTful item Id.

  • SKU - This is a free form string created by the seller for an item, for example V-00031-WHM
  • ItemID - This is the Id of the listing, which can be for a specific item or a group of items, for example 110039490209.

If the SKU field is returned.

This indicates this listing is for a specific item within a group. To get the RESTful item Id of the item, pass in the ItemID and SKU values in using the Get Item by Legacy Id call in the Browse API.

browse/v1/item/get_item_by_legacy_id?legacy_item_id=110039490209&legacy_variation_sku=V-00031-WHM

If the SKU field is not returned.

This indicates this listing is for a specific item, which in not part of a group. To get the RESTful item Id, pass in the itemID value using the Get Item by Legacy Id call in the Browse API.


browse/v1/item/get_item_by_legacy_id?legacy_item_id=110039490209

Finding API item Ids

The Finding API response includes the following fields. You use these fields to determine which Browse API call to use to get the RESTful item Id.

  • isMultiVariationListing - This is an boolean value that indicates whether the listing is for a listing with multiple variations.
  • itemId - This is the Id of the listing, for example 262446562346.
isMultiVariationListing = true

This indicates this listing is a group of items with variations. So the itemId value represents an item group. To get the Ids for each item in the group, pass in the itemId value using the Get Item by Item Group call in the Browse API.


browse/v1/item/get_items_by_item_group?item_group_id=262446562346

isMultiVariationListing = false

This indicates this listing is for a specific item. To get the RESTful item Id, pass in the itemId value using the Get Item by Legacy Id call in the Browse API.


browse/v1/item/get_item_by_legacy_id?legacy_item_id=262446562346
