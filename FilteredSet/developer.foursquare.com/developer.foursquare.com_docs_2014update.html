Foursquare API August 2014 Update

We hope you’re as excited about our recent release of Foursquare 8.0 as we are. If you don’t already have it, download it for iOS or Android right now and start playing around with a brand-new personalized search and discovery experience.

But what does this launch mean for you as a Foursquare developer? The API documented on this site continues to be the exact same API that both Foursquare 8.0 and Swarm use. Over the next few months we’ll be making some announcements about changes and new features to the API that you should prepare for.

Deprecating Legacy Features

Although some features exposed in our API are already no longer part of the Foursquare and Swarm experiences, our goal with any API transition is to give developers enough time to react and adapt their integrations. However, we’re officially announcing that developers should no longer be relying on product features that were part of Foursquare 7.0 and earlier, including badges, the leaderboard, and certain special types. Since these won’t be part of the Foursquare experience moving forward, you should migrate away from using them in your own experiences you build on top of Foursquare.

We are giving developers until the end of 2014 to migrate away from these features and endpoints. Our API Endpoint overview page will be up-to-date by the end of the week to reflect endpoints that will be deprecated by the end of the year. In 2015, these endpoints will no longer be supported and may stop functioning without any notice.

Changes to Versioning

Over the past year, we have began enforcing our API versioning rules more, and moving forward versioning is only going to become more important. Moving forward, there will be a new required parameter for API versions v >= 20140806 .

As we begin to evolve two new user experiences in Foursquare and Swarm, it sometimes makes sense for the same API endpoint to return different pieces of information depending on if you want the “Foursquare perspective” (e.g., see tips in the responses) or the “Swarm perspective” (e.g., see info about check-ins in the responses). Our new m (for “mode”) parameter lets developers specify whether they want Foursquare- or Swarm-style responses by passing in m=foursquare or m=swarm .

Our documentation will soon reflect what endpoints support what m parameters. When you next update the versions you send in your API requests, please add the appropriate m param value to your request. API responses for versions v < 20140806 will remain the same.

New Features

We’re almost ready to release some of the awesome new features that are powering our new apps to you guys and will be releasing documentation in the next few weeks for our new recommendations engine, user profile aspects (tastes and expertise), and expiring tip creation. Stay tuned and be sure to follow us on Twitter for the latest! If you have any specific new features you’d like to request, let us know there as well.

Authentication with Two Apps

Of course we still offer the ability for Foursquare users to connect their accounts to your app so you can do things such as create check-ins or tips on their behalf, see their venue histories, and more. For now, our OAuth process remains exactly the same, and developers and apps will be able to see a connected user’s information across both Foursquare and Swarm (although we have two apps, there is still only one User model for Foursquare users).

Native auth will continue to be supported in Foursquare 8.0, and more importantly, the interface to both the iOS and Android libraries have not changed. We expect the majority of Swarm users to also have Foursquare installed, but implementing a fallback to the web view-based implementation of auth has always been recommended as well, especially to account now for Swarm users that may not have Foursquare installed.

Branding for Foursquare & Swarm

To avoid confusion for your users, please replace all references to "Foursquare check-ins" with "Swarm check-ins." You should also refresh any imagery or icons you have with our newly-released assets . Long live and !

Things We Didn't Mention Here

If you’re wondering “will the release of Foursquare and Swarm affect my application?” and the particular issue you’re wondering about isn’t mentioned here, it’s likely that the answer to your question is “not at all!”
