SpecialHours

This type is used to express the special operating hours of a merchant's store on a specific date. A specialHours container is needed when the store's opening hours on a specific date are different than the normal operating hours on that particular day of the week.

Note: This type is not yet applicable since establishing In-Store Pickup or Click and Collect inventory is not supported with the the initial release of the Inventory API.

Types that use SpecialHours:

    InventoryLocation , InventoryLocationFull , InventoryLocationResponse

Calls that use SpecialHours:

    /location GET , /location/{merchantLocationKey} POST , /location/{merchantLocationKey}/update_location_details POST

Fields

date ( string ) [0..?]
Required if specialHours container is used. This is a specific date for which this set of special hours apply. The date is actually expressed as a timestamp specified in ISO 8601 format, which uses the 24-hour Coordinated Universal Time (UTC) clock. The following examples show (1) the format of the time-stamp, and (2) an example time value in ISO 8601 format:

Format: YYYY-MM-DDTHH:MM:SS.SSSZ
Example: 2016-10-19T00:09:00.000Z

Returned if set for the store location.
See the Field Index to learn which calls use date.

intervals ( array of Interval ) [0..?]
Required if you specify specialHours for the location. This container is used to define the opening and closing times of the store location for the date specified in the date field. These hours will override the standard operating hours for that particular day of the week.

Returned if set for the store location.
See the Field Index to learn which calls use intervals.
