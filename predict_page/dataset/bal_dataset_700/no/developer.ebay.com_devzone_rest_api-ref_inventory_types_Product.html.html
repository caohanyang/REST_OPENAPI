Product

This type is used to define the product details, such as a title, a product description, item specifics, and links to images for the product. Optionally, in a Create or Replace Inventory Item call, a seller can pass in a Global Trade Identification Number (GTIN) value such as an EAN, an ISBN, a UPC, or a Brand and Manufacturer Part Number (MPN) pair to identify a product to be matched with a product in the eBay catalog. The information in this type is also returned in the two Get calls.

Type that uses Product:

    InventoryItem

Calls that use Product:

    /inventory_item GET , /inventory_item/{sku} PUT

aspects ( array of string ) [0..?]
This is an array of item specific pairs that provide more information about the product. These item specifics pairs are automatically populated when a seller specifies a Global Trade Item Number (GTIN) or Brand/MPN pair and a product match is found in the eBay product catalog. If a GTIN is not used, the seller can input item specific pairs in this container.

See the example below to get an idea of the syntax that is used to specify common aspects: 

"aspects": {
 "Brand": ["GoPro"],
 "Storage Type": ["Removable"]
 }
See the Field Index to learn which calls use aspects.

brand ( string ) [0..?]
Required if a Manufacturer Part Number (MPN) value will be used to identify a motor vehicle part. The brand of the product. This field is paired with the MPN field to identify a part/component for a motor vehicle. eBay will search for the motor vehicle part/accessory in the eBay product catalog, and if a match is found, the inventory item is automatically populated with available product details such as a title, a product description, item specifics, and links to stock images for the motor vehicle part.
See the Field Index to learn which calls use brand.

description ( string ) [0..?]
Required if a product identifier is not being used to match a product in the eBay product catalog. The description of the product. This description of the inventory item is automatically populated if and when a matching product is found in the eBay product catalog. The description of an existing inventory item can be added or modified with a Create or Replace Inventory Item call.

Note that this field is product description field, and not a listing description field, although they may be similar, especially in the case of a single-variation listing. The listing description field is actually passed in the Create Offer call. If the inventory item will be part of an inventory item group/multiple-variation listing, this field should definitely be used to specify how the corresponding product variation is different (e.g. This is the green, extra-large version of the shirt ).
See the Field Index to learn which calls use description.

ean ( array of string ) [0..?]
The European Article Number/International Article Number (EAN) for the product. This field can be used to identify a product. eBay will search for the product in the eBay product catalog, and if a match is found, the inventory item is automatically populated with available product details such as a title, a product description, item specifics, and links to stock images for the product.
See the Field Index to learn which calls use ean.

imageUrls ( array of string ) [0..?]
Required if a product identifier is not being used to match a product in the eBay product catalog. An array of one or more links to images for the product. URLs can be "HTTP" or "HTTPS". If available, links to stock images of the inventory item are automatically populated if and when a matching product is found in the eBay product catalog. The exact links appearing for an existing inventory item can be changed with a Create or Replace Inventory Item call.
See the Field Index to learn which calls use imageUrls.

This field can be used to identify a product. eBay will search for the product in the eBay product catalog, and if a match is found, the inventory item is automatically populated with available product details such as a title, a product description, item specifics, and links to stock images for the product.

mpn ( string ) [0..?]
Required if a Manufacturer Part Number (MPN) value will be used to identify a motor vehicle part. The Manufacturer Part Number (MPN) of a motor vehicle part. This field is paired with the brand field to identify a part/component for a motor vehicle. eBay will search for the motor vehicle part in the eBay product catalog, and if a match is found, the inventory item is automatically populated with available product details such as a title, a product description, item specifics, and links to stock images for the motor vehicle part.
See the Field Index to learn which calls use mpn.

subtitle ( string ) [0..?]
The listing subtitle that will be used if product is listed on eBay. The subtitle of an existing inventory item can added, modified, or removed with a Create or Replace Inventory Item call. Note that there is a small fee for including a subtitle with a listing on eBay.

Note that the same subtitle text should be used for each inventory item that will be part of an inventory item group, and ultimately become one product variation within a multipe-variation listing.
See the Field Index to learn which calls use subtitle.

title ( string ) [0..?]
Required if a product identifier is not being used to match a product in the eBay product catalog. The listing title that will be used if product is listed on eBay. This title for the inventory item is automatically populated if and when a matching product is found in the eBay product catalog. The title of an existing inventory item can be added or modified with a Create or Replace Inventory Item call.

Note that the same title text should be used for each inventory item that will be part of an inventory item group, and ultimately become one product variation within a multipe-variation listing.
See the Field Index to learn which calls use title.

upc ( array of string ) [0..?]
The Universal Product Code (UPC) value for the product. This field can be used to identify a product. eBay will search for the product in the eBay product catalog, and if a match is found, the inventory item is automatically populated with available product details such as a title, a product description, item specifics, and links to stock images for the product.
