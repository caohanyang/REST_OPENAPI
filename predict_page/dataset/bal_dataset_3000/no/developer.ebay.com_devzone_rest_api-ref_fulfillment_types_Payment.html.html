Payment

Contains information about a payment made for an order.

Type that uses Payment:

    PaymentSummary

Calls that use Payment:

    /order GET , /order/{orderId} GET

Fields

amount ( Amount )
paymentDate ( string )
paymentHolds ( array of PaymentHold )
paymentMethod ( PaymentMethodTypeEnum )
paymentReferenceId ( string )
paymentStatus ( PaymentStatusEnum )


amount ( Amount ) [0..?]
The monetary amount of the order.
See the Field Index to learn which calls use amount.

paymentDate ( string ) [0..?]
The date and time that the payment was received by the seller. The timestamp is in ISO 8601 format, which uses the 24-hour Universal Coordinated Time (UTC) clock.

Format: YYYY-MM-DDTHH:MM:SS.SSSZ
Example: 2015-08-04T19:09:02.768Z
See the Field Index to learn which calls use paymentDate.

paymentHolds ( array of PaymentHold ) [0..?]
Contains information about a hold (if any) placed on some or all of the payment, including the reason why the payment for the order is being held, the expected release date of the funds into the seller's account, the current state of the hold, and if the payment has been released, the actual release date. This container is returned only if a hold exists.
See the Field Index to learn which calls use paymentHolds.

paymentMethod ( PaymentMethodTypeEnum ) [0..?]
The method by which the payment was made.
See the Field Index to learn which calls use paymentMethod.

paymentReferenceId ( string ) [0..?]
Returned only if the value of paymentMethod is PAYPAL . This field contains the PayPal-generated transaction identifier.
See the Field Index to learn which calls use paymentReferenceId.

paymentStatus ( PaymentStatusEnum ) [0..?]
The degree of completion of the payment.
See the Field Index to learn which calls use paymentStatus.
