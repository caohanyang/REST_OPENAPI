ErrorDetailV3

This type is used to express detailed information on errors and warnings that may occur with a call request.

Types that use ErrorDetailV3:

    BaseResponse , FeeSummary , OfferResponse , PriceQuantityResponse , PublishResponse

Calls that use ErrorDetailV3:

    /bulk_update_price_quantity POST , /inventory_item/{sku} PUT , /inventory_item/{sku}/product_compatibility PUT , /inventory_item_group/{inventoryItemGroupKey} PUT , /offer POST , /offer/get_listing_fees POST , /offer/{offer_Id} PUT , /offer/{offerId}/publish POST

Fields

category ( string )
domain ( string )
errorId ( integer )
inputRefIds ( array of string )
longMessage ( string )
message ( string )
outputRefIds ( array of string )
parameters ( array of ErrorParameterV3 )
subdomain ( string )


category ( string ) [0..?]
This string value indicates the error category. There are three categories of errors: request errors, application errors, and system errors.
See the Field Index to learn which calls use category.

domain ( string ) [0..?]
The name of the domain in which the error or warning occurred.
See the Field Index to learn which calls use domain.

errorId ( integer ) [0..?]
A unique code that identifies the particular error or warning that occurred. Your application can use error codes as identifiers in your customized error-handling algorithms.
See the Field Index to learn which calls use errorId.

inputRefIds ( array of string ) [0..?]
An array of one or more reference IDs which identify the specific request element(s) most closely associated to the error or warning, if any.
See the Field Index to learn which calls use inputRefIds.

longMessage ( string ) [0..?]
A detailed description of the condition that caused the error or warning, and information on what to do to correct the problem.
See the Field Index to learn which calls use longMessage.

message ( string ) [0..?]
A description of the condition that caused the error or warning.
See the Field Index to learn which calls use message.

outputRefIds ( array of string ) [0..?]
An array of one or more reference IDs which identify the specific response element(s) most closely associated to the error or warning, if any.
See the Field Index to learn which calls use outputRefIds.

parameters ( array of ErrorParameterV3 ) [0..?]
Various warning and error messages return one or more variables that contain contextual information about the error or waring. This is often the field or value that triggered the error or warning.
See the Field Index to learn which calls use parameters.

subdomain ( string ) [0..?]
The name of the subdomain in which the error or warning occurred.
See the Field Index to learn which calls use subdomain.
