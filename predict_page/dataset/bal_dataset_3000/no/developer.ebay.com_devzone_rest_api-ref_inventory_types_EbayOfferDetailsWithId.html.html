EbayOfferDetailsWithId

This type provides details of an offer, and is used by the base request payload of the Update Offer call. Every field that is currently set with the unpublished/published offer must also be passed into the Update Offer call, even those fields whose values are not changing.

Call that uses EbayOfferDetailsWithId:

    /offer/{offer_Id} PUT

Fields

availableQuantity ( integer )
categoryId ( string )
listingDescription ( string )
listingPolicies ( ListingPolicies )
merchantLocationKey ( string )
pricingSummary ( PricingSummary )
quantityLimitPerBuyer ( integer )
storeCategoryNames ( array of string )
tax ( Tax )


availableQuantity ( integer ) [0..?]
This integer value will update the quantity of the inventory item (specified by the sku value) that will be available for purchase by buyers shopping on the eBay site specified in the marketplaceId field. This field is required even if the available quantity is not changing for the offer.
See the Field Index to learn which calls use availableQuantity.

categoryId ( string ) [0..?]
The unique identifier of the eBay category that the inventory item is listed under. This field is required even if the category ID is not changing for the offer.
See the Field Index to learn which calls use categoryId.

listingDescription ( string ) [0..?]
The description of the product. This field is required even if the listing description is not changing for the offer.
See the Field Index to learn which calls use listingDescription.

listingPolicies ( ListingPolicies ) [0..?]
This container is used to set the listing policies that will be used to construct the listing. This container consists of the payment, return, and fulfillment listing policies that will be associated with the eBay listing. This container is required even if the listing policies are not changing for the offer. It is required that the seller be opted in to Business Policies before being able to create live eBay listings through the Inventory API. Sellers can opt-in to Business Policies through My eBay or by using the Account API's Opt-in to Program call. Similarly, payment, return, and fulfillment listing policies may be created/managed in My eBay or by using the listing policy calls of the Account API.
See the Field Index to learn which calls use listingPolicies.

merchantLocationKey ( string ) [0..?]
The unique identifier of an inventory location (where the inventory item in the offer is located). A merchantLocationKey value is established when the merchant creates an inventory location using the Create Inventory Location call. This value should remain the same as the value passed in with the Create Offer call.

Max length : 36
See the Field Index to learn which calls use merchantLocationKey.

pricingSummary ( PricingSummary ) [0..?]
This container shows the listing price for the product offer, and if applicable, the settings for the Minimum Advertised Price and Strikethrough Pricing features. The Minimum Advertised Price feature is only available on the US site. Strikethrough Pricing is available on the US, eBay Motors, UK, Germany, Canada (English and French), France, Italy, and Spain sites. This field is required even if the pricing information is not changing for the offer.
See the Field Index to learn which calls use pricingSummary.

quantityLimitPerBuyer ( integer ) [0..?]
Required if this field was included with the original Create Offer call. This field is only applicable and set if the seller wishes to set a restriction on the purchase quantity per seller. If this field is set by the seller for the offer, then each distinct buyer may purchase up to, but not exceeding the quantity specified for this field. So, if this field's value is 5 , each buyer may purchase between one to five of these products, and the purchases can occur in one multiple-quantity purchase, or over multiple transactions. If a buyer attempts to purchase one or more of these products, and the cumulative quantity will take the buyer beyond the quantity limit, that buyer will be blocked from that purchase. If this field was used in the original Create Offer call, it must also be included in an Update Offer call, even if the seller is not modifying the value.
See the Field Index to learn which calls use quantityLimitPerBuyer.

storeCategoryNames ( array of string ) [0..?]
Required if this container was included with the original Create Offer call. This container is used if the seller would like to place the inventory item into one or two store categories that the seller has set up for their store. The string value(s) passed in to this container will be the full path(s) to the store categories, as shown below: 
"storeCategoryNames": [
 "/Fashion/Men/Shirts", 
 "/Fashion/Men/Accessories" ],

If this field was used in the original Create Offer call, it must also be included in an Update Offer call, even if the store categories are not changing.
See the Field Index to learn which calls use storeCategoryNames.

tax ( Tax ) [0..?]
Required if this container was used in the original Create Offer call, even if the store categories are not changing. This container is only applicable and used if a sales tax table, a Value-Added Tax (VAT) rate, or a tax exception category code will be applied to the offer. Only Business Sellers can apply VAT to their listings. It is possible that the applyTax field will be included with a value of true , but a buyer's purchase will not involve sales tax. A sales tax rate must be set up in the seller's sales tax table for the buyer's state/tax jurisdiction in order for that buyer to be subject to sales tax.
See the Field Index to learn which calls use tax.
