Metric

A complex type that defines the metrics on which sellers are evaluated.

Type that uses Metric:

    StandardsProfile

Calls that use Metric:

    /seller_standards_profile GET , /seller_standards_profile/{program}/{cycle} GET

Fields

level ( StandardsLevelEnum )
lookbackEndDate ( string )
lookbackStartDate ( string )
metricKey ( string )
name ( string )
thresholdLowerBound ( any )
thresholdMetaData ( string )
thresholdUpperBound ( any )
type ( DataTypeEnum )
value ( any )


level ( StandardsLevelEnum ) [0..?]
Indicates the seller level for this metric.
See the Field Index to learn which calls use level.

lookbackEndDate ( string ) [0..?]
The end date of the period during which the seller was evaluated for the associated metric.
See the Field Index to learn which calls use lookbackEndDate.

lookbackStartDate ( string ) [0..?]
The beginning date of the period during which the seller was evaluated for the associated metric.
See the Field Index to learn which calls use lookbackStartDate.

metricKey ( string ) [0..?]
The name of the metric. For example, Defect rate .
See the Field Index to learn which calls use metricKey.

name ( string ) [0..?]
The complete name of the metric for which the seller is being evaluated. This value is localized according to the value of the X-EBAY-C-MARKETPLACE-ID request header.
See the Field Index to learn which calls use name.

thresholdLowerBound ( any ) [0..?]
The lower limit of the metric threshold used in the evaluation. Each program can have different threshold levels.
See the Field Index to learn which calls use thresholdLowerBound.

thresholdMetaData ( string ) [0..?]
Expression for the type of metric, and the inclusive and exclusive characteristics of the upper and lower bounds of the threshold. Thresholds are shown for either a count or a rate.
For example, the following metadata means that to maintain the level for this metric, you must continue to have a value of 90 or better, and there is no upper bound for this metric:
[90, NA)
Metadata values are enclosed by parentheses and/or square brackets. A parenthesis means the adjacent value is exclusive, while a square bracket means the adjacent value is inclusive.
See the Field Index to learn which calls use thresholdMetaData.

thresholdUpperBound ( any ) [0..?]
The upper limit of the metric threshold used in the evaluation. Each program can have different threshold levels.
See the Field Index to learn which calls use thresholdUpperBound.

type ( DataTypeEnum ) [0..?]
Indicates the data type of the returned metric, such as a string or integer value.
See the Field Index to learn which calls use type.

value ( any ) [0..?]
The seller's value, or score, for the metric.
See the Field Index to learn which calls use value.
