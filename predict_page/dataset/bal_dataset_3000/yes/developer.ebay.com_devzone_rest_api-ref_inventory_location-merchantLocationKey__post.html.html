Create Inventory Location

POST /location/{merchantLocationKey}

Use this call to create a new inventory location. Specify the seller-defined, immutable merchant location key using the call's path parameter. If successful, this call returns the URI to the newly created inventory-location sub-resource in the Location response header.

Note that all locations are "enabled" by default when they are created, and you must specifically disable them if you want them to be hidden from location searches.

Sample: Create a location

Creates an inventory location. Each inventory item must be assigned an inventory location.

Description

Use this call to configure an inventory location that sellers will use as the source of one or more inventory items.

Input

Suppy a unique seller-defined merchantLocationKey as a path parameter in this call. This particular location is a warehouse, as indicated in the locationTypes field. The address of the warehouse is provided in the address container and user-friendly name of the location is specified in the name field. The merchantLocationStatus field is included and set to ENABLED to allow active inventory items to be associated with this location. 

URL format. See also the non-wrapped version of this URL.

POST https://api.ebay.com/sell/inventory/v1/location/warehouse-1

{
    "location": {
        "address": {
            "addressLine1": "2055 Hamilton Ave",
            "addressLine2": "Building 3",
            "city": "San Jose",
            "stateOrProvince": "CA",
            "postalCode": "95125",
            "country": "US"
        }
    },
    "locationInstructions": "Items ship from here.",
    "name": "Warehouse-1",
    "merchantLocationStatus": "ENABLED",
    "locationTypes": [
        "WAREHOUSE"
    ]
}

Output

There is no response payload for this call. 

JSON format.
If successful, this call returns an empty payload and an HTTP status code of "204 No Content".
