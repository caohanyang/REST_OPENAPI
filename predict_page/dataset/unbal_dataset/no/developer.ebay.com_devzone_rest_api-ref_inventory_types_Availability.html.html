Availability

This type is used to specify the quantity of the inventory item that is available for purchase if the item will be shipped to the buyer, and the quantity of the inventory item that is available for In-Store Pickup or Click and Collect at one or more of the merchant's physical stores. In-Store Pickup is only available to large merchants selling on the US, UK, Germany, and Australia sites. Click and Collect is only available in the UK and Australia, but there are future plans to roll this feature out to the Germany site.

Note: The pickupAtLocationAvailability container is not yet available for use. In the future, this container will allow sellers to set the quantity of the inventory item that are available for In-Store Pickup or Click and Collect at one or more of the merchant's physical stores.

Type that uses Availability:

    InventoryItem

Calls that use Availability:

    /inventory_item GET , /inventory_item/{sku} PUT

Fields

pickupAtLocationAvailability ( array of PickupAtLocationAvailability ) [0..?]
This container consists of an array of one or more of the merchant's physical store locations where the inventory item is available for In-Store Pickup or Click and Collect orders. The merchant's location, the quantity available, and the fulfillment time (how soon the item will be ready for pickup after the order takes place) are all in this container.

Note: The pickupAtLocationAvailability container is not yet available for use. In the future, this container will allow sellers to set the quantity of the inventory item that are available for In-Store Pickup or Click and Collect at one or more of the merchant's physical stores.
See the Field Index to learn which calls use pickupAtLocationAvailability.

shipToLocationAvailability ( ShipToLocationAvailability ) [0..?]
This container specifies the quantity of the inventory item that are available for a standard purchase where the item is shipped to the buyer.

Note: Although this container is technically optional, until the The pickupAtLocationAvailability container is actually enabled, this container will be required when using the Create or Replace Inventory Item call.
See the Field Index to learn which calls use shipToLocationAvailability.
