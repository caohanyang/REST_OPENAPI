InventoryCriterion

Type that defines the fields for item IDs for the promotion. These IDs are either the seller's item IDs or the eBay listing IDs.

Types that use InventoryCriterion:

    ItemPromotion , ItemPromotionResponse

Calls that use InventoryCriterion:

    /item_promotion POST , /item_promotion/{promotion_id} GET

Fields

inventoryCriterionType ( InventoryCriterionEnum ) [0..?]
Indicates if the promotion is being created using rules or a list of inventory. Note: Currently, only using a list of inventory IDs is supported.

Required unless the promotion is in DRAFT state.

See the Field Index to learn which calls use inventoryCriterionType.

inventoryItems ( array of InventoryItem ) [0..?]
An array of containers for the seller's inventory reference IDs to be added to the promotion.

Note: The request can have only one of these: inventoryItems or listingIds . Also, all listing in a promotion must offer PayPal as a payment method. Maximum: 500 parent items
Maximum SKU or custom label length: 50 characters

Required unless the promotion is in DRAFT state.




See the Field Index to learn which calls use inventoryItems.

listingIds ( array of string ) [0..?]
An array of eBay listing IDs list to be added to the promotion.

Note: The request can have only one of these: inventoryItems or listingIds . Also, all listing in a promotion must offer PayPal as a payment method. Maximum: 500 parent items



Required unless the promotion is in DRAFT state.

See the Field Index to learn which calls use listingIds.
