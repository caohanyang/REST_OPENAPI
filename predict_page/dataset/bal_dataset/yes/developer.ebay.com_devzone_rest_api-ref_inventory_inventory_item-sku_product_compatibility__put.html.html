Create or Replace Product Compatibility

PUT /inventory_item/{sku}/product_compatibility

This call is used by the seller to create or replace a list of products that are compatible with the inventory item. The inventory item is identified with a SKU value in the URI. Product compatibility is currently only applicable to motor vehicle parts and accessory categories, but more categories may be supported in the future.

In addition to the authorization header, which is required for all eBay REST API calls, the following table includes another request header that is mandatory for the Create or Replace Product Compatibility call, and another request header that is optional:


Header Description Required? Applicable Values
Content-Language Use this header to control the language that is used for any returned errors or warnings in the call response. Yes en-US
Content-Type The MIME type of the body of the request. Must be JSON. No application/json

Request field descriptions



Input Container/Field Type Occurrence Meaning
compatibleProducts array of CompatibleProduct Required This container consists of an array of motor vehicles (make, model, and year) that are compatible with the motor vehicle part or accessory specified by the sku value.
compatibleProducts. notes string Conditional This field is optionally used by the seller to input any notes pertaining to the parts compatibility being specified. The seller might use this field to specify the placement of the part on a vehicle or other applicable information. This field will only be returned if specified by the seller.
compatibleProducts
. productFamilyProperties 		ProductFamilyProperties Conditional This container consists of an array of motor vehicles that are compatible with the motor vehicle or part or accessory specified by the sku value. These motor vehicle are identified by properties such as make, model, year, trim, and engine type.

Either the productIdentifier or the productFamilyProperties container (or both) can be used to specify one or more motor vehicles that are compatible with the inventory item. If the productIdentifier container is used, the product identifiers that are passed in that container must match vehicles in the eBay catalog in order for that vehicle to be added to the compatible vehicle list.
compatibleProducts
.productFamilyProperties
. engine 		string Conditional The specifications of the engine, including its size, block type, and fuel type. An example is 2.7L V6 gas DOHC naturally aspirated .
compatibleProducts
.productFamilyProperties. make 		string Required The make of the vehicle (e.g. Toyota ).
compatibleProducts
.productFamilyProperties. model 		string Required The model of the vehicle (e.g. Camry ).
compatibleProducts
.productFamilyProperties. trim 		string Conditional The trim of the vehicle (e.g. 2-door Coupe ).
compatibleProducts
.productFamilyProperties. year 		string Required The year of the vehicle (e.g. 2016 ).
compatibleProducts
. productIdentifier 		ProductIdentifier Conditional This container is used in a Create Or Replace Product Compatibility call to identify a motor vehicle that is compatible with the inventory item. If the motor vehicle is found in the eBay catalog, the motor vehicle properties (the properties in the productFamilyProperties ) will automatically be populated for the vehicle.

Either the productIdentifier or the productFamilyProperties container (or both) can be used to specify one or more motor vehicles that are compatible with the inventory item. If the productIdentifier container is used, the product identifiers that are passed in that container must match vehicles in the eBay catalog in order for that vehicle to be added to the compatible vehicle list.
compatibleProducts
.productIdentifier. epid 		string Optional This field can be used if the seller already knows the eBay product ID associated with the motor vehicle that is to be added to the compatible product list. If this eBay product ID is found in the eBay product catalog, the motor vehicle properties (engine, make, model, trim, and year) will automatically get picked up for that motor vehicle.
compatibleProducts
.productIdentifier. gtin 		string Optional This field can be used if the seller knows the Global Trade Identification Number for the motor vehicle that is to be added to the compatible product list. If this GTIN value is found in the eBay product catalog, the motor vehicle properties (engine, make, model, trim, and year) will automatically get picked up for that motor vehicle.
compatibleProducts
.productIdentifier. ktype 		string Optional This field can be used if the seller knows the K Type Number for the motor vehicle that is to be added to the compatible product list. If this K Type value is found in the eBay product catalog, the motor vehicle properties (engine, make, model, trim, and year) will automatically get picked up for that motor vehicle.

Only the DE, UK, and AU sites support the use of K Type Numbers.
sku string Optional This is the seller-defined SKU value of the inventory item that will be associated with the compatible vehicles. This field is not applicable to the Create or Replace Product Compatibility call, but it is always returned with the Get Product Compatibility call. For the the Create or Replace Product Compatibility call, the SKU value for the inventory item is actually passed in as part of the call URI.
