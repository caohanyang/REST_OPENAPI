Get Campaign by ID

GET /ad_campaign/{campaign_id}


This call retrieves the specified campaign using the ID. You pass in the campaign ID as a URI parameter.

Input Output Error Codes Samples Change History

Input

See also Samples .

Resource URI (production) 

GET https://api.ebay.com/sell/marketing/v1/ad_campaign/{campaign_id}

URI parameters

Parameter Type Required? Meaning
campaign_id string Required Identifier of a campaign in which to retrieve. This ID was generated when the campaign was created.

campaignCriterion
. autoSelectFutureInventory 		boolean Conditionally If set to true, all future inventory matching the criterion will be added to the campaign. This means when a new listing is added to eBay that is a Promoted Listings eligible item (fixed price and has multiple quantity) and matches the selection rules of a campaign, it will be automatically added to the campaign.

Default: false

INVENTORY_PARTITION
Indicates the criterion is based on inventory reference IDs. These will be either the eBay listing ID or a seller's inventory reference ID.

Code so that your app gracefully handles any future changes to this list.

. selectionRules 		array of SelectionRule Conditionally Set of selection rules used to select the listings in the campaign.
  • Each set of selection rules are ORed with each other.
  • Individual rules within a selection rule set are ANDed with each other. If a rule has a list of values, such a list of category IDs, the item must match only one of the values of the rule.
    • Note: If an item matches multiple sets of rules or multiple rules within a selection rule set, the item is considered only once.

fundingStrategy FundingStrategy Conditionally The container for the funding strategy (bid percentage and funding model) of the campaign.
fundingStrategy. bidPercentage string Conditionally The default bid percentage for the funding model. This is the percentage of the sale price that the seller will be charged by eBay when the listing sells. If bid percentage is not provided, the default bid percentage associated with the campaign is used. The higher the bid, the better the position of the listing on the page.

The bid percentage is a single precision value. For example, 4.1, 4.10, 5.5, 5.0, etc. are valid values and 4.44, 8.11, etc. would be invalid values.

Minimum value: 1.0
Maximum value: 20




fundingStrategy. fundingModel string Conditionally Indicates the funding model associated with the campaign.

Default: COST_PER_SALE.

Applicable values are from FundingModelEnum :

COST_PER_SALE
Indicates the seller is charged for a promoted listing when the listing sells.

Code so that your app gracefully handles any future changes to this list.

Applicable values are from MarketplaceIdEnum :See marketplaceId .
Code so that your app gracefully handles any future changes to this list.
startDate string Conditionally The start date of the campaign. On this day, the service derives the keywords for each listing, creates an ad for each listing as associates the ad with the campaign and then starts the campaign. The amount of time it takes the service to start the campaign depends on the number of listings in the campaign. You can check it the campaign has started by using the Get Campaign call.

Valid Format: yyyy-MM-ddThh:mm.ss.sssZ

Sample: Retrieve a Specific Campaign by ID

Retrieves the specified campaign by its ID.

Description

This call returns the details for the specified campaign.

Input

The input for this call is the campaign_id as a URI parameter. 

URL format. See also the non-wrapped version of this URL.

GET https://api.ebay.com/sell/marketing/v1/ad_campaign/10000176012

Output

The output is just the basic information about this campaign because there are no ads associated with the campaign. 

JSON format.
{
  "campaignId": "10000176012",
  "startDate": "2016-08-30T05:28:32.000Z",
  "endDate": null,
  "campaignName": "Summer Sale",
  "fundingStrategy": {
    "fundingModel": "COST_PER_SALE",
    "bidPercentage": "1.5"
  },
  "campaignStatus": "RUNNING",
  "marketplaceId": "EBAY_US"
}
