Create Ads By Inventory Reference

POST /ad_campaign/{campaign_id}/create_ads_by_inventory_reference


This call creates an ad for the specified inventory reference and associates the ad with the specified campaign. You specify the campaign ID has a URI parameter and the bidPercentage , inventoryReferenceId , and inventoryReferenceType of the listing. Each listing becomes an ad that is associated with the campaign.

Note: If the inventory reference is for a variation, an ad is created for each variation. For example, a shirt that comes in 3 sizes and 3 colors, would result in 9 ads. Keep this in mind so that you stay under the 25K maximum listings per campaign limit.

Payload model

The following lists all fields that could be included in the request. 

{ /* CreateAdsByInventoryReferenceRequest */
"bidPercentage": string,
"inventoryReferenceId": string,
"inventoryReferenceType": string
}

Request field descriptions



Input Container/Field Type Occurrence Meaning
bidPercentage string Required The bid percentage associated with the seller's inventory reference. This is the percentage of the sale price that the seller will be charged by eBay when the listing sells. If bid percentage is not provided, the default bid percentage associated with the campaign is used. The higher the bid, the better the position of the listing on the page.

The bid percentage is a single precision value. For example, 4.1, 4.10, 5.5, 5.0, etc. are valid values and 4.44, 8.11, etc. would be invalid values.

Minimum value: 1.0
Maximum value: 20




inventoryReferenceId string Required The seller's inventory reference ID for a listing. This can be an ID of a listing or the ID of the parent of a listing with variations (a group of items), such as a shirt that is available in multiple sizes and colors. You must always pass in both inventory_reference_id and inventory_reference_type .

Maximum IDs per call: 500

inventoryReferenceType string Required Indicates the inventory reference ID type of the inventoryReferenceId . You must always pass in both inventory_reference_id and inventory_reference_type .

Applicable values are from InventoryReferenceTypeEnum :

INVENTORY_ITEM
Indicates the inventory reference ID is for a listing.
INVENTORY_ITEM_GROUP
Indicates the inventory reference ID is the parent of a listing with variations (a group of items), such as a shirt that is available in multiple sizes and colors.

HTTP status codes

This call can return one of the following HTTP status codes. See the HTTP Status Code Registry for a complete overview of HTTP status codes.

Status Meaning
201 The call was successful and the ad was created.
400 A HTTP status code of 400 indicates there is an error in the request. Refer to the documentation for information about the request.
404 The campaign ID submitted does not exist.
500 A HTTP status code of 500 indicates there was a problem with an eBay internal service. Wait a few minutes and try the call again.

Sample: Create Ad Using an Inventory Reference ID

Creates an ad for a specific inventory reference ID and associates it with the specified campaign.

Description

This sample creates an ad for the listing with the inventory reference ID of 87536492 and associates the ad with the campaign having the ID of 10001741014.

Input

The inputs for this sample are campaign_id as a URI parameter and the inventory_reference_id and inventory_reference_type of the listing and the bidPercentage for the ad. 

URL format. See also the non-wrapped version of this URL.

POST https://api.ebay.com/sell/marketing/v1/ad_campaign/10001741014/create_ads_by_inventory_reference
{
   "bidPercentage":"10",
   "inventoryReferenceId":"87536492",
   "inventoryReferenceType":"INVENTORY_ITEM"
}

Output

The output is the ID and URL of the ad. 

JSON format.
{
   "ads":[
      {
         "adId":"4536625344",
         "href":"https://api.ebay.com/sell/marketing/v1/ad_campaign/10001741014/ad/4536625344"
      }
   ]
}
