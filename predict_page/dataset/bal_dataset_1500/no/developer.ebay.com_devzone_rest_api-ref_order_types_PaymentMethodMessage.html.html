PaymentMethodMessage

Type that defines the fields for legal messages and buyer consent verification.

Type that uses PaymentMethodMessage:

    PaymentMethod

Calls that use PaymentMethodMessage:

    /checkout_session/initiate POST , /checkout_session/{checkoutSessionId} GET , /checkout_session/{checkoutSessionId}/update_payment_info POST , /checkout_session/{checkoutSessionId}/update_quantity POST , /checkout_session/{checkoutSessionId}/update_shipping_address POST , /checkout_session/{checkoutSessionId}/update_shipping_option POST , /guest_checkout_session/initiate POST , /guest_checkout_session/{checkoutSessionId} GET , /guest_checkout_session/{checkoutSessionId}/update_payment_info POST , /guest_checkout_session/{checkoutSessionId}/update_quantity POST , /guest_checkout_session/{checkoutSessionId}/update_shipping_address POST , /guest_checkout_session/{checkoutSessionId}/update_shipping_option POST

Fields

legalMessage ( string ) [0..?]
Information that eBay is legally obligated to show to the buyer. This field can be null, in which case do nothing. But if this field is not null, the value of this field must appear on the checkout page. Note: This field is used only for non-US purchases.
See the Field Index to learn which calls use legalMessage.

requiredForUserConfirmation ( boolean ) [0..?]
Indicates if you need to get consent from the buyer before proceeding with the checkout. If this value is true, you must include a consent checkbox on the checkout page. In order to proceed with the checkout, the buyer must click the box.
See the Field Index to learn which calls use requiredForUserConfirmation.
