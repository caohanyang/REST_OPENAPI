CampaignCriterion

Type that defines the fields for the criterion settings.

Types that use CampaignCriterion:

    Campaign , CreateCampaignRequest

Calls that use CampaignCriterion:

    /ad_campaign POST , /ad_campaign/find_campaign_by_ad_reference GET , /ad_campaign/get_campaign_by_name GET , /ad_campaign/{campaign_id} GET

Fields

autoSelectFutureInventory ( boolean ) [0..?]
If set to true, all future inventory matching the criterion will be added to the campaign. This means when a new listing is added to eBay that is a Promoted Listings eligible item (fixed price and has multiple quantity) and matches the selection rules of a campaign, it will be automatically added to the campaign.

Default: false

See the Field Index to learn which calls use autoSelectFutureInventory.

criterionType ( CriterionTypeEnum ) [0..?]
Enumeration type that defines the criterion types. Currently only INVENTORY_PARTITION is supported.
See the Field Index to learn which calls use criterionType.

selectionRules ( array of SelectionRule ) [0..?]
Set of selection rules used to select the listings in the campaign.
  • Each set of selection rules are ORed with each other.
  • Individual rules within a selection rule set are ANDed with each other. If a rule has a list of values, such a list of category IDs, the item must match only one of the values of the rule.
    • Note: If an item matches multiple sets of rules or multiple rules within a selection rule set, the item is considered only once.
Maximum number: 10
See the Field Index to learn which calls use selectionRules.
