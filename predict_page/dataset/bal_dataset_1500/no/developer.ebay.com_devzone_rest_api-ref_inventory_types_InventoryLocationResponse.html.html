InventoryLocationResponse

This type is used by the base response of the Get Inventory Location and Get Inventory Locations calls. These responses provide details about inventory location(s) defined for the merchant's account.

Type that uses InventoryLocationResponse:

    LocationResponse

Calls that use InventoryLocationResponse:

    /location GET , /location/{merchantLocationKey} GET

Fields

location ( Location )
locationAdditionalInformation ( string )
locationInstructions ( string )
locationTypes ( array of StoreTypeEnum )
locationWebUrl ( string )
merchantLocationKey ( string )
merchantLocationStatus ( StatusEnum )
name ( string )
operatingHours ( array of OperatingHours )
phone ( string )
specialHours ( array of SpecialHours )


location ( Location ) [0..?]
This container provides location details of an inventory location. The address container will always be returned, but it will not always have a complete street address. Except in the case of an inventory location that supports In-Store Pickup or Click and Collect inventory, a full address is not a requirement when setting up an inventory location. The geoCoordinates container will only be set if the merchant provided geographical coordinates.
See the Field Index to learn which calls use location.

locationAdditionalInformation ( string ) [0..?]
This text field provide additional information about an inventory location. This field is returned if it is set for the inventory location.

Max length : 256
See the Field Index to learn which calls use locationAdditionalInformation.

locationInstructions ( string ) [0..?]
This text field provide special pickup instructions for the store location. This field can help create a pleasant and easy pickup experience for In-Store Pickup and Click and Collect orders. If this field was not set up through a Create Inventory Location or a Update Inventory Location call, eBay will use the default pickup instructions contained in the merchant's profile.

Note: This container is not yet applicable since establishing In-Store Pickup or Click and Collect inventory is not supported with the the initial release of the Inventory API.
Max length : 1000
See the Field Index to learn which calls use locationInstructions.

locationTypes ( array of StoreTypeEnum ) [0..?]
This container defines the function of the inventory location. Typically, an inventory location will serve as a store or a warehouse, but in some cases, an inventory location may be both.

The location type of an inventory location defaults to WAREHOUSE if a location type is not specified when a merchant creates an inventory location.
See the Field Index to learn which calls use locationTypes.

locationWebUrl ( string ) [0..?]
This text field shows the Website address (URL) for the store location. This field is returned if defined for the inventory location.

Max length : 512
See the Field Index to learn which calls use locationWebUrl.

merchantLocationKey ( string ) [0..?]
The unique identifier of the inventory location. This identifier is set up by the merchant when the inventory location is first created with the Create Inventory Location call. Once this value is set for an inventory location, it can not be modified.

Max length : 36
See the Field Index to learn which calls use merchantLocationKey.

merchantLocationStatus ( StatusEnum ) [0..?]
This field indicates whether the inventory location is enabled (inventory can be loaded to location) or disabled (inventory can not be loaded to location). The merchant can use the Enable Inventory Location call to enable an inventory location in disabled status, or the Disable Inventory Location call to disable an inventory location in enabled status.
See the Field Index to learn which calls use merchantLocationStatus.

name ( string ) [0..?]
The name of the inventory location. This name will generally be a human-friendly name as it will be displayed in user interfaces.

Max length : 1000
See the Field Index to learn which calls use name.

operatingHours ( array of OperatingHours ) [0..?]
This container shows the regular operating hours for a store location during the days of the week. A dayOfWeekEnum field and an intervals container is shown for each day of the week that the store location is open.

Note: This container is not yet applicable since establishing In-Store Pickup or Click and Collect inventory is not supported with the the initial release of the Inventory API.
See the Field Index to learn which calls use operatingHours.

phone ( string ) [0..?]
The phone number for an inventory location. This field will typically only be set and returned for store locations.

Max length : 36
See the Field Index to learn which calls use phone.

specialHours ( array of SpecialHours ) [0..?]
This container shows the special operating hours for a store location on a specific date or dates.

Note: This container is not yet applicable since establishing In-Store Pickup or Click and Collect inventory is not supported with the the initial release of the Inventory API.
See the Field Index to learn which calls use specialHours.
